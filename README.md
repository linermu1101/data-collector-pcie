# 多卡PCIe太赫兹信号采集系统

## 项目简介

本项目是一个基于Qt框架的多卡PCIe数据采集系统，主要用于太赫兹信号的高速采集与实时可视化。系统采用双管线架构设计，实现了数据采集管线与控制/监测管线的分离，通过异步信号机制与非阻塞策略，确保在高通量数据采集场景下的稳定性和响应效率。

本系统面向核聚变等离子体参数检测、太赫兹源信号系统等高通量实时可视化场景，适用于工业检测、智能交通、生物医学、环境监测与科研教学等领域。

## 主要特性

### 核心功能
- **多卡并发采集**：支持多达13张PCIe采集卡同时工作，实现高通量数据采集
- **实时可视化**：时域/频域信号实时显示，支持FFT频谱分析
- **双触发模式**：支持软件触发和硬件触发两种采集模式
- **DDR缓存采集**：支持大容量DDR缓存数据采集，适用于长时间离线采集
- **实时流采集**：支持实时数据流采集与显示
- **数据存储**：支持DAT/INF格式的数据文件存储

### 架构特点
- **双管线分离**：数据采集管线与控制/监测管线独立运行，互不阻塞
- **异步解耦**：采用Qt信号/槽机制实现模块间异步通信
- **背压监测**：实时监测FIFO填充率，提供分级预警机制
- **线程优先级管理**：采集线程高优先级，确保数据采集连续性
- **非阻塞设计**：提示与日志处理不阻塞采集主通路

### 预警与容错
- **阈值分层提示**：基于FIFO填充率的分级预警（常态/预警/重要）
- **溢出检测**：实时检测FIFO溢出事件
- **日志限频**：防止日志风暴影响系统性能
- **异常恢复**：设备异常快速复位路径

- FFT频谱分析

- 分级日志记录
- 日志轮转管理
- 参数持久化存储
- 运行状态记录

## 配置说明

主要配置项位于 `config.ini` 文件：

### 基本参数 [BASIC_PARA]
```ini
WorkMode=三频              # 工作模式
Pahse_PickUp=100us         # 相位拾取
Phase_Filp=自动判断         # 相位翻转
HardTrigCard=1             # 硬件触发主卡
Coefficient=1              # 系数
DebugMode=false            # 调试模式
SimulatedCardCount=0       # 模拟卡数量
```

### 网络参数 [NET_PARA]
```ini
LocalPort=8080             # 本地端口
Feekback_Way=自动判断       # 反馈方式
serverReflectionPath=...   # 服务器反射路径
```

### 通道参数 [CHANNEL_PARA]
- 支持最多13张采集卡
- 每张卡支持多个通道配置
- 可配置通道存储开关

### 寄存器参数 [RegisterParams]
```ini
nf0_r=21                   # 参考信号第一极值点
nf1_r=75                   # 参考信号第二极值点
nf2_r=97                   # 参考信号第三极值点
change_rt=12500            # 反馈间隔点数
singal_mode=1              # 单频/三频选择
```

## 使用说明

### 设备连接
1. 启动程序后，点击"连接设备"按钮
2. 系统自动扫描PCIe设备并建立连接
3. 状态指示灯显示连接状态

### 数据采集
1. **实时采集**：选择实时模式，设置采样参数，点击"开始"
2. **离线采集**：选择DDR模式，设置采集时间，点击"开始"
3. **硬件触发**：配置硬件触发参数，等待外部触发信号

### 参数配置
1. 点击"参数配置"按钮打开配置面板
2. 设置极值参数、采样率、触发模式等
3. 配置自动保存到config.ini

### 预警处理
- 系统自动监测FIFO状态
- 状态栏显示预警信息
- 高占用时自动降低刷新率
- 溢出时记录事件日志

## 扩展与定制

### 添加新的采集卡
1. 在config.ini中添加新卡配置
2. 配置通道参数和存储选项
3. 系统自动识别并加载

### 自定义预警策略
1. 修改阈值分层参数
2. 调整采样周期
3. 配置提示级别映射

### 扩展信号处理
1. 在signalprocessing模块添加处理算法
2. 通过信号/槽机制接入数据流
3. 更新显示窗口

## 注意事项

1. **硬件要求**：确保PCIe设备驱动正确安装
2. **资源预留**：根据卡数量预留足够的CPU和内存资源
3. **日志存储**：建议将日志输出定向至独立磁盘分区
4. **线程优先级**：不要随意修改采集线程优先级
5. **数据备份**：定期备份配置文件和日志
